class LinearHistory(val baseVersionHash: String?)

Was introduced in https://github.com/modelix/modelix/commit/19c74bed5921028af3ac3ee9d997fc1c4203ad44 together with the UndoOp. The idea is that an undo should only revert changes if there is no other change that relies on it. In that case the undo should do nothing, to not indirectly undo newer changes. For example, if you added a node and someone else started changing properties on the that node, your undo should not remove the node to not lose the property changes. This requires the versions to be ordered in a way that the undo appears later.
